Signed Integer Overflow

Undefined behavior (UB-CCV1).

According to ISO C11, if the mathematical of an arithmetic operation producing a signed result can not be represented in the result type the behavior is undefined. In practice, this error can cause buffer overflow vunlerabilites. Because it is allowed for the minimum value in a signed type to have a greater absolute value than the maximum value, it is also possible for a prefix negation or a division (by -1) to produce an integer overflow.

Citations

see C11 section 6.5:5
see C11 section J.2:1 item 36
see CERT-C section INT32-C

Division By Zero

Undefined behavior (UB-CEMX1).

According to ISO C11, if the value of the second operand of a division expression is zero, the behavior is undefined. In practice, this error is very serious and may trigger abnormal termination of the entire application. Input data should be compared against zero before division is performed.

Citations

see C11 section 6.5.5:5
see C11 section J.2:1 item 45
see CERT-C section INT33-C
